■ IT

・IT
情報技術の総称。

・ICT
情報通信技術の略。
ほぼITと同じだが、海外ではこちらが一般的。

・IoT
モノがインターネット経由で通信すること。


■ OS

・OS
システム全体を管理し、多様なアプリケーションソフトを動かすための最も基本的ソフトウェア。

・Windows
MicrosoftのOS。最もメジャー。

・macOS (Mac OS X)
AppleのOS。同じAppleの製品と相性が良い。

・UNIX
古いOS。多くのOSの派生元でもある。

・Linux
無料のOS。改変や配布が自由に行える。
リーナスさんがUNIXを真似して作ったのでLinux。



■ 言語・環境

・マークアップ
WEBページの大まかなデザインを作る言語のこと。

・フロントエンド
WEBページの動作を作る言語。あるいはマークアップも含めた言語のこと。

・HTML
WEBページに文字を出力し、基本的なレイアウトを加えるマークアップ言語。

・CSS
WEBページのレイアウトを独自のものに変えるスタイルシート言語。

・JavaScript
指定したユーザー側の動作に対し、予め設定した様々な反応を返したりできるスクリプト言語。

・サーバーサイド
サーバー側で実行されるプログラムを設計する言語のこと。

・.NET（C#, VB）
Windows系のアプリを作成する際に用いられる言語。

・Java
様々な分野のプログラムを作ることができる言語。

・Java SE
Javaの基本機能をまとめたもの。

・Java EE（Jakarta EE）
JavaSEを元にしてサーバーサイドの機能を追加したもの。

・Perl
テキスト処理に向いている言語。

・PHP
Webシステムの開発に向いている言語。
他言語に比べ、HTMLを作成するのではなく埋め込むのが特徴的。

・Ruby
Webシステムの開発に向いている言語。
汎用性と自由度が高く、記述量が少ないため、PHPよりも大規模なシステム作成に長けている。


■ サーバーソフト

・Apache
動画を中心としたWEBサイトに向いている。

・nginx
Apacheに比べて高機能だが、大量の動的コンテンツには不向き。


・IIS
マウス操作が可能で比較的使いやすい。
PHPやjavaで開発されたものを動かすには向かない。


■ SQL

・SQL
データベースを操作するための言語。
「シークェル」または「シーケル」。

・DB2
リレーショナルデータベース管理システムの総称。
使用データベース言語はSQL。

・MySQL
Webとの親和性が高く、レンタルサーバーを利用してWeb開発を目指すエンジニアにも向いている。
無償で処理能力が早い。

・Maria DB
基本的にMySQLと扱い方は同じだが、MySQLよりも高機能な面が多い。

・Oracle DB
サポートが厚いため安定性が高く、複雑な作業に向いている。
難易度が高く、高価。

・PostgreSQL
MySQLよりも複雑な作業ができるが、速度において劣っている。

・SQLite
開発時によく使うデータベース。
ロックする範囲をデータベース全体でしか制御できず、また複数のサーバーと接続するのは不向き。

・SQL Server
Windows系のOSと相性がよく、マウスによる作業が可能。

・CRUD
データベースの処理を表す造語。
create、read、update、deleteの頭文字。在庫管理でイメージしやすい。

・トランザクション
複数の処理を1つにまとめたもの。

・コミット
トランザクション処理の結果を成功確定させること。

・ロールバック
データ更新などで障害が起こったときに、その前の状態にまで戻すこと。

・排他制御（排他ロック）
ファイルへの書き込みが行われている間、別のプログラムによる書き込みを禁止する制御のこと。

・楽観ロック
データそのものに対してロックは行わず、更新対象のデータが取得時と同じ状態であることを確認してから更新すること。

・悲観ロック
更新対象のデータを取得する際にロックをかけることで、他のトランザクションから更新されないようにすること。ロックが解除不能になったときの事前対応策が必須。

・実行計画
より短い時間で無駄なくSQLを実行する方法をまとめたもの。

・チューニング
既存システムの処理短縮や負荷軽減を目的とした調整作業。

・パフォーマンス
性能の度合い。

・スロークエリ
実行時間が指定時間を超過したSQL。

・クエリログ
クライアントからのMySQL Serverへの接続・接続解除の情報、およびクライアントから実行された全てのSQLクエリを出力した履歴情報。


IDE（統合開発環境）

・Eclipse

プラグインが非常に豊富。

・NetBeans

・DBツール

・A5:SQL Mk-2

・pgAdmin
PostgreSQLの開発ツール。

・DBeaver
様々なデータベースに対応した開発ツール。

・MySQL Workbench
MySQLをGUIでわかりやすく操作する開発ツール。

・Sequel Pro
MySQLをGUIでわかりやすく操作するMac用の開発ツール。

・TeamSQL
多くのデータベースに対応した開発ルール。
現在はサポートが終了している。


■ 肩書

・PM
開発プロジェクトの責任者。
プロジェクトを立上げ、必要な全ての準備と調達を行う。

・PL
プロジェクトを円滑に進めるための指示と管理を行う。

・PG（PE）
SEが作成した設計書に沿ってプログラミングを行う。

・SE
プログラムの設計を行い、全体の動作管理を行う。

・デザイナー
Webサイトの外見上のデザインを決める。

・エンジニア
様々な種類があり、主に設計・開発・運用を行う。

・フロントエンド・エンジニア
ユーザーが見たり、操作したりするWEBページ上のコーディングを担当する。

・サーバーサイド・エンジニア
サーバーの設計や構築、また運用や保守などを担当する。

・バックエンド・エンジニア
Web領域を中心としたサーバー側でのプログラム開発を担当する。


■ UI（ユーザーインターフェース）

・UI
ユーザーが使用する上での操作性。

・CUI
情報を文字によって表示し、文字によって入力するUI。

・GUI
情報を画面上のウィンドウやアイコンとして表示し、マウスクリックなどによって操作できるUI。


■ ファイル／ディレクトリ

・ファイル
コンピュータにおけるデータの管理単位。

・拡張子
ファイルの種類や形式を表す文字列。

・ディレクトリ
ファイルがあるデータの保管場所。フォルダのこと。

・ルートディレクトリ
ファイル階層における最初または最上位のディレクトリ。

・カレントディレクトリ
現時点で作業を行なっているディレクトリ。

・パス
ファイルやディレクトリの位置を表すもの。

・絶対パス
ルートディレクトリからの位置関係を表記したもの。

・相対パス
カレントディレクトリから指定のファイルやディレクトリへの位置関係を表記したもの。


■ ネットワーク

・LAN
限られた範囲内にあるコンピュータや通信機器などに接続し、相互にデータ通信できるようにしたネットワーク。

・WAN
市街地から郊外、県外、海外に広がった広域のネットワークのこと。

・クライアント
サービスや機能を使う側のコンピュータ。

・サーバー
サービスを提供している企業側のコンピュータ。


■ 各種プロトコル

・HTTP
サーバとクライアントの間でデータの送受信を行うために用いられるプロトコル。

・FTP
ファイル転送を行うためのプロトコル。

・HTTPS
HTTPが暗号化されている状態のプロトコル。

・SSL
データを暗号化して送受信するためのプロトコル。

・SSH
通信経路を暗号化して別のコンピュータを操作するためのソフトウェア。

・ポート、ポート番号
コンピュータ内で動作する複数のソフトウェアのどれが通信するかを指定するための番号。

・HTTP、FTP、SSHの各ポート番号
HTTP = 80　FTP = 20(データ)/21(制御)　SSH = 22

・ファイアウォール
外部の攻撃から内部を保護するためのソフトウェア。

・TCP
速度よりも安全性を重視する通信プロトコル。


■ IPアドレス

・IPアドレス
ネットワークに接続されたコンピュータや通信機器の一台ごとに割り当てられた識別番号

・グローバルIPアドレス
インターネットで相手と通信するための、他のアドレスと重複しないのIPアドレス。

・プライベートIPアドレス
LANなどの企業内ネットワークに存在する端末が、自由に使用できるIPアドレス。


■ ドメイン

・ドメイン
インターネット上の住所。
URLならばhttps://以降の文字列が該当する。

・DNS
ドメイン名 を管理・運用するためのシステム。
IPアドレスをドメイン名という形で分かりやすくする。

・VPN
他所でも専用回線のように安全な通信を実現する技術。

・ブリッジ
ネットワークにおける中継機器。
交差点の都度に案内するカーナビみたいなもの。

・プロキシサーバー
使用コンピューターに代わってインターネットに接続し、アクセスなどを行うサーバー。


■ 電子メール

・SMTP

・POP

・IMAP

■ バージョン管理

・Git
